Understanding PartialFC: Training 10 Million Identities on a Single Machine

PartialFC at a Glance
PublishedCVPR 2022
Scale10M+ identities
Key IdeaSample partial class centers per step
Speedup~40% less communication
PaperarXiv:2203.15565
CodeInsightFace

The Problem

Modern face recognition models need to distinguish between millions of identities. The classifier layer alone — a weight matrix of shape C × d where C can reach 10 million — becomes the dominant memory and communication bottleneck during distributed training.

With standard model parallelism, every training step requires an AllGather of the full logit vector (size C) across all GPUs, followed by a synchronized softmax. As C grows into the millions, this becomes prohibitively expensive.

The Core Idea

PartialFC's insight is simple: you don't need all class centers at every step. For a given mini-batch, only a tiny fraction of classes are "positive" (i.e., have samples in the batch). The remaining negative classes can be subsampled without significantly affecting convergence.

Concretely, each GPU:

  1. Keeps its positive class centers (the ones matching labels in the current batch)
  2. Randomly samples a small subset of negative class centers
  3. Computes logits against only this reduced set

The softmax denominator is computed over the sampled subset rather than the full C classes. Empirically, sampling as few as 10% of negatives produces nearly identical accuracy to using all classes.

Why It Works

The gradient of the softmax loss for a negative class is proportional to its predicted probability. For most negatives, this probability is vanishingly small — they contribute almost nothing to the gradient. By sampling, we keep the important gradients (positives + hard negatives that happen to be sampled) while saving enormous amounts of computation and communication.

Communication Savings

The key win is in the backward pass. Instead of AllReduce over a gradient matrix of shape C × d, PartialFC only synchronizes gradients for the sampled subset — typically 10–30% of C. This directly reduces communication volume by 70–90%.
