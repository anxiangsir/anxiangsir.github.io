1) 引子：扩散模型到底在做什么？1) Introduction: What does a diffusion model do?

核心问题很朴素：如何从纯噪声生成有结构的数据。扩散方法把这个难题拆成很多个简单步骤——先把真实样本逐步加噪，再学习“逆向去噪”把噪声拉回数据流形。The core challenge is simple: generate structured data from pure noise. Diffusion breaks this into easy steps—gradually add noise to real data, then learn the reverse denoising process back to the data manifold.

为什么重要？因为它稳定、可扩展、生成质量高；缺点是经典采样太慢。于是有了 DDIM（同模型更快采样）和 Flow Matching（更直接的 ODE 学习框架）。Why it matters: stable training, scalable quality, and strong generation; downside: classic sampling is slow. That motivates DDIM (faster sampling with the same model) and Flow Matching (a cleaner ODE-style learning framework).

2) DDPM：基础范式2) DDPM: the foundation

前向过程：每一步注入少量高斯噪声，最终得到接近标准高斯的 xT。Forward process: inject a little Gaussian noise each step until xT is near standard Gaussian.

DDPM Forward
q(xt|xt-1) = N(√(1-βt)xt-1, βtI)
xt = √(ᾱt)x0 + √(1-ᾱt)ε

反向过程：训练神经网络 εθ(xt, t) 预测噪声，再一步步去噪：Reverse process: train εθ(xt, t) to predict noise, then denoise step by step:

L = E[ || ε - εθ(xt, t) ||² ]
关键洞察：理论扎实，但常见采样约 1000 步，慢。Key insight: theoretically grounded, but ~1000 sampling steps are slow.

3) DDIM：同一模型的确定性捷径3) DDIM: deterministic shortcut with same model

DDIM 不改训练，只改采样轨迹。把随机项设为 0（σt=0）后，路径变成确定性映射，可大步跳时刻。DDIM keeps the same training and changes only sampling trajectories. Set stochasticity to zero (σt=0), get deterministic mapping, and skip timesteps aggressively.

DDIM Sampling Update
xt-1 = √(ᾱt-1)·x̂0 + √(1-ᾱt-1)·direction
实践常见：50 步、20 步即可生成可用样本。In practice: 50-step or even 20-step sampling often works well.

4) Flow Matching：优雅的速度场视角4) Flow Matching: elegant velocity-field view

Flow Matching 直接学习一个 ODE 速度场：点在 t∈[0,1] 上沿向量场流动。最常见设置是线性插值轨迹：Flow Matching directly learns an ODE velocity field: points flow along vectors on t∈[0,1]. A common setup uses straight interpolation:

xt = (1-t)x0 + tε
vtarget = ε - x0
L = E[ || vθ(xt, t) - (ε - x0) ||² ]

这和 Rectified Flow、Stable Diffusion 3 背后的思想一脉相承：在更规则的路径上学速度，采样可用常规 ODE 求解器完成。This aligns with ideas behind Rectified Flow and Stable Diffusion 3: learn velocity on more regular paths, then sample with standard ODE solvers.

# DDPM Training
def ddpm_train_step(model, x_0):
    t = randint(0, T)
    noise = randn_like(x_0)
    x_t = sqrt(alpha_bar[t]) * x_0 + sqrt(1 - alpha_bar[t]) * noise
    predicted_noise = model(x_t, t)
    loss = mse(noise, predicted_noise)
    return loss

# DDPM Sampling (slow)
def ddpm_sample(model):
    x = randn(shape)  # pure noise
    for t in reversed(range(T)):  # T=1000 steps!
        z = randn_like(x) if t > 0 else 0
        predicted_noise = model(x, t)
        x = (1/sqrt(alpha[t])) * (x - (1-alpha[t])/sqrt(1-alpha_bar[t]) * predicted_noise) + sigma[t] * z
    return x
# DDIM Sampling (fast, same model!)
def ddim_sample(model, steps=50):
    x = randn(shape)
    timesteps = linspace(T, 0, steps)  # skip steps!
    for t, t_prev in zip(timesteps[:-1], timesteps[1:]):
        predicted_noise = model(x, t)
        predicted_x0 = (x - sqrt(1-alpha_bar[t]) * predicted_noise) / sqrt(alpha_bar[t])
        x = sqrt(alpha_bar[t_prev]) * predicted_x0 + sqrt(1-alpha_bar[t_prev]) * predicted_noise
    return x
# Flow Matching Training
def flow_matching_train_step(model, x_0):
    t = rand()  # uniform [0,1]
    noise = randn_like(x_0)
    x_t = (1 - t) * x_0 + t * noise  # straight interpolation!
    velocity = noise - x_0  # target velocity
    predicted_v = model(x_t, t)
    loss = mse(velocity, predicted_v)
    return loss

# Flow Matching Sampling
def flow_matching_sample(model, steps=50):
    x = randn(shape)  # pure noise at t=1
    dt = 1.0 / steps
    for i in range(steps):
        t = 1 - i * dt
        v = model(x, t)
        x = x - v * dt  # Euler step, straight path
    return x

一句话总结：DDPM 给了最稳健的概率基础，DDIM 让它真正“可用”，Flow Matching 则把“从噪声到数据”的过程改写成更简洁的速度场学习问题。One-line takeaway: DDPM provides the probabilistic foundation, DDIM makes it practical, and Flow Matching reframes generation as clean velocity-field learning.
