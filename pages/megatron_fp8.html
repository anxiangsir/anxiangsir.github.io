1. 为什么要 FP8 训练？

训练大型语言模型的主要瓶颈之一是显存（HBM）带宽和容量。传统混合精度训练（AMP）使用 BF16/FP16 做前向与反向，FP32 存储 master weights，内存开销巨大。FP8 训练把激活值和权重梯度压缩到 8 位，可将相关计算的带宽需求减半，同时 H100 GPU 上 FP8 Tensor Core 的峰值算力是 BF16 的两倍。

🔴 传统 AMP 训练

  • 权重：BF16（前向）+ FP32（master）
  • 梯度：FP32 累积
  • 激活：BF16
  • 每个参数 6 字节（BF16 权重 + FP32 master）

🟢 FP8 混合精度训练

  • 权重：FP8 E4M3（前向 GEMM）
  • 权重梯度：FP8 E5M2
  • 激活：FP8（计算时）
  • master weights / 优化器状态仍用 FP32
📦 类比： 把货物用更小的集装箱运输——计算的"货物"（张量数值）被更紧凑地编码，运输速度（带宽利用率）提升，但需要精确的装箱规范（缩放因子）防止溢出或精度损失。

2. FP8 数据格式：E4M3 与 E5M2

FP8 标准（IEEE 草案 OFP8）定义了两种 8 位浮点格式，各有侧重：

格式 符号位 指数位 (E) 尾数位 (M) 最大值 典型用途
E4M3FN143 448 前向激活、权重
E5M2152 57344 梯度（范围更大）
BF16187 ~3.4×1038 传统 AMP

8 位浮点数的编码方式与 IEEE 754 单精度类似，只是位宽更窄：

\[ \text{value} = (-1)^{s} \cdot 2^{e - \text{bias}} \cdot (1 + m / 2^M) \]

其中 \(s\) 是符号位，\(e\) 是指数字段的无符号整数值，\(\text{bias}\) 是格式常数（E4M3 中 bias=7，E5M2 中 bias=15），\(m\) 是尾数整数值，\(M\) 是尾数位数。

🔑 为什么前向用 E4M3、反向用 E5M2？
前向传播的激活和权重数值通常集中在一个较小的范围内，精度（更多尾数位）比动态范围更重要 → E4M3（3 位尾数）更合适。反向传播的梯度数值分布往往更稀疏、峰值更大，动态范围（更多指数位）比精度更重要 → E5M2（2 位尾数，5 位指数）更合适。

3. Tensor 缩放因子

FP8 的最大表示值只有 448（E4M3）或 57344（E5M2），远小于 BF16 的 3.4×10³⁸。因此，直接把 BF16 张量转成 FP8 几乎必然溢出或下溢。解决方案是为每个张量维护一个缩放因子 \(s\)：

\[ x_{\text{fp8}} = \text{cast\_fp8}\!\left(\frac{x}{s}\right), \qquad \hat{x} = x_{\text{fp8}} \cdot s \]

理想的缩放因子应使量化后的张量充分利用 FP8 的动态范围，即将张量的绝对最大值（amax）映射到 FP8 最大值附近：

\[ s = \frac{\text{amax}(x)}{\text{fp8\_max}} \]

延迟缩放（Delayed Scaling）

实时计算 amax 需要在 GEMM 之前扫描整个张量，带来额外开销。Megatron-LM（通过 transformer_engine）使用"延迟缩放"策略：维护一个历史窗口的 amax 记录，用上一步（或前几步）的 amax 来估算当前步的缩放因子。

即时缩放（Just-in-time）

每步计算当前 amax 并立即更新 s。精度最高，但需要在 GEMM 前做一次归约操作（额外 kernel 启动开销）。

延迟缩放（Delayed）

用前 N 步（默认 1 步）的 amax 来预测本步的 s。避免额外归约，几乎零额外开销。数值稳定性同样很好，是 Megatron 的默认策略。

4. 前向传播与反向传播

在 Transformer 的线性层中，FP8 训练的核心是三个 GEMM（矩阵乘法）操作，每个操作使用不同的精度组合：

\[ \underbrace{Y = X W^T}_{\text{前向 (E4M3×E4M3)}} \quad \underbrace{dX = dY W}_{\text{输入梯度 (E5M2×E4M3)}} \quad \underbrace{dW = dY^T X}_{\text{权重梯度 (E5M2×E4M3)}} \]
输入 X
(BF16)
→

(→ E4M3)
→

(H100 TC)
→
输出 Y
(BF16)

累加器（GEMM 的输出）仍以 FP32 或 BF16 精度保存，以避免精度损失的累积。仅计算密集型的矩阵乘法使用 FP8，LayerNorm、SoftMax、残差连接等操作仍在 BF16 下执行。

⚖️ 关键洞察： FP8 训练不是"把所有东西都换成 FP8"，而是有针对性地把最耗带宽的矩阵乘法换成 FP8，其余保持高精度。这就像只在高速公路段用小集装箱，在装卸时仍用标准托盘——兼顾速度与安全。

6. Megatron-LM 中的 FP8 实践

Megatron-LM 通过 NVIDIA transformer_engine 库提供 FP8 支持，只需几行配置即可启用。transformer_engine 内部自动管理缩放因子、amax 历史、以及 FP8 到 BF16 的转换。

# pretrain_gpt.py / 启动脚本中添加以下参数
--fp8-format hybrid      # E4M3 前向 + E5M2 反向
--fp8-amax-history-len 16  # amax 历史窗口大小
--fp8-amax-compute-algo max  # 取历史最大值作为缩放依据

# Python API (transformer_engine)
import transformer_engine.pytorch as te
from transformer_engine.common.recipe import Format, DelayedScaling

fp8_recipe = DelayedScaling(
    fp8_format=Format.HYBRID,       # E4M3 fwd / E5M2 bwd
    amax_history_len=16,
    amax_compute_algo="max",
)

with te.fp8_autocast(enabled=True, fp8_recipe=fp8_recipe):
    output = model(input_ids, attention_mask)

Megatron-LM 中的 FP8 兼容张量并行（Tensor Parallel）和流水线并行（Pipeline Parallel），每个 GPU 的激活 amax 通过 all-reduce 在 TP 组内同步，确保各卡的缩放因子一致。

💡 实践建议：
  • 使用 --fp8-format hybrid（E4M3 前向 + E5M2 反向）而非 e5m2，精度更好。
  • amax 历史窗口设为 16–32，取 max 策略在动态变化的训练曲线上更稳定。
  • 首次启用 FP8 时建议先用少量步骤验证 loss 曲线与 BF16 基线一致，再扩大规模。
  • 梯度裁剪（grad clip）阈值可略微放宽（如 1.0→1.5），因为 FP8 梯度的量化噪声可能使 norm 偏大。
  • 不支持 FP8 的操作（如 FlashAttention v2）会自动回退到 BF16，无需手动处理。

7. 性能基准

以下数据来自 NVIDIA Megatron-LM 官方报告（H100 SXM 80GB，GPT 类模型，TP=8，PP=1）：

配置 模型规模 精度 吞吐量 (tokens/s/GPU) 相对加速
Baseline7BBF16~4,2001.0×
FP8 Hybrid7BFP8+BF16~6,1001.45×
Baseline70BBF16~5301.0×
FP8 Hybrid70BFP8+BF16~8801.66×

模型规模越大，带宽瓶颈越显著，FP8 的加速效果越好（70B 优于 7B）。在内存方面，激活检查点（activation checkpointing）配合 FP8 激活存储，可将激活内存降低至 BF16 基线的约 40–50%。

📊 模型质量： 在标准语言建模基准（Pile、MMLU、HellaSwag 等）上，FP8 训练与 BF16 基线的 loss 曲线和下游任务指标高度吻合（差异 < 0.1%），验证了 FP8 在大规模训练中的数值可靠性。
