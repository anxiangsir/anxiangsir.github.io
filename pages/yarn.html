YaRN

Yet another RoPE extensioN method

大型语言模型（LLM）的上下文窗口正在极速扩张。
YaRN 提供了一种优雅且计算零开销的方法，
通过动态调整旋转位置编码（RoPE），将上下文长度扩展数倍。

1. RoPE 旋转位置编码基础

在理解 YaRN 之前，我们必须先理解 RoPE。RoPE 将词向量的元素两两分组，并在复平面上进行旋转，旋转的角度与该词在句子中的绝对位置 m 成正比。

f(x, m, θd) = eimθd x

更严谨地，将其写为复数形式的旋转（在 2D 子空间中）：

fW(xm, m, θd) = eimθd W xm

后续的所有上下文扩展方法（包括 YaRN），本质上都是通过引入位置函数 g(m) 和频率函数 h(θ) 来修改这个旋转公式：

f'W(xm, m, θd) = fW(xm, g(m), h(θd))

关键在于，不同维度 d 拥有不同的旋转频率 θd：
低维度（小 d）旋转极快，捕捉局部的相对距离；
高维度（大 d）旋转极慢，几乎静止，编码宏观的绝对位置信息。

θd = 10000-2d/|D|
d=0 (Fast)
d=8
d=16
d=24 (Slow)
m = 0

2. 波长与上下文长度

旋转频率 θd 决定了该维度在序列中需要多少个 Token 才能完成一次完整的 2π 旋转。我们将这个跨度称为波长 (Wavelength) λd。

λd = 2π / θd = 2π · 100002d/|D|

定义预训练上下文长度 L 与波长的比值 r(d)：

r(d) = L / λd = L / (2π · 100002d/|D|)

这个比值决定了该维度如何被处理：当 r ≫ 1 时，该维度在训练窗口内完成多次循环 → 高频（局部信息）；当 r ≪ 1 时，该维度几乎不旋转 → 低频（全局位置信息）。

预训练上下文长度 (L) 4096

3. 线性插值 (PI) 的灾难

要扩展上下文到 s·L，最直接的方法是 Position Interpolation (PI)：把所有位置坐标缩小为 m/s。在频域上，这等于把所有频率压缩为 θd/s。

但这对高频维度（低 d）是灾难性的。高频维度原本用来区分非常近的 Token，将它们硬性拉伸后，相邻 Token 的相对角度差变得过小，导致模型“近视”，丢失了细粒度的相对位置分辨能力。

用数学公式表达，PI 相当于：

g(m) = m/s,    h(θd) = θd

这等价于将所有维度的有效频率变为 θd/s。在数学上，原本相邻 Token 间的旋转角度差是 θd，现在变成了 θd/s。对于高频维度来说，角度差变得极小，导致模型“近视”，丢失了细粒度的相对位置分辨能力。

缩放比例 (s) 1x

4. NTK-Aware 插值 (前置基础)

在引入分段 NTK 之前，我们先看看 NTK-Aware Interpolation。它不直接缩放位置，而是修改底数 b，将插值的压力分散到所有维度上：

b' = b · s|D|/(|D|-2)

高频维度受影响较小，低频维度受影响较大。但这依然不够完美，于是 YaRN 提出了 NTK-by-parts。

4.1 YaRN 核心：分段 NTK 插值

YaRN 的精髓在于 NTK-by-parts：不要一刀切地缩放。引入比值 r = L / λd，并定义两个阈值 α=1, β=32：

■ 高频 (r > β)：波长远小于L。完全不插值，保持原样。
■ 中频 (α < r < β)：使用斜坡函数平滑过渡。
■ 低频 (r < α)：波长大于L。执行 PI 线性插值 (除以s)。

h(θd) = (1-γ(r)) · θd/s + γ(r) · θd

这里的 γ(r) 是一个斜坡函数（Ramp function），其具体定义为：

γ(r) = 0,                if r < α
γ(r) = 1,                if r > β
γ(r) = (r-α)/(β-α),   otherwise

让我们用具体数字（以 LLaMA 为例，L=4096, b=10000）来看三个极端情况：

1. 最高频 (d=0)：θ0 = 1.0，波长 λ0 ≈ 6.28。比值 r ≈ 652 ≫ β。此时 γ=1，完全使用原频率 θ0，保留局部信息。
2. 最低频 (d=31)：θ31 极小，波长远大于 L。比值 r ≈ 0.00065 ≪ α。此时 γ=0，使用全量 PI 插值 θ31/s。
3. 中频：按比例平滑混合。

目标扩展倍数 (s) 16x

4.5. 动态 NTK 与直觉 (Intuition)

在实际推理时，我们并不知道最终序列长度。YaRN 使用动态缩放因子，根据当前序列长度 l' 实时计算：

s = max(1, l'/L)

这意味着模型可以在不进行任何微调（Fine-tuning）的情况下，在推理时动态适应不同的上下文长度。

为什么分段 NTK 如此有效？（Intuition）
核心洞察是：高频维度编码的是局部距离（相邻 Token）。它们的波长极短，在预训练窗口 L 内已经历了无数个完整周期。它们已经见过了所有可能的相对位置，如果强行插值，反而会破坏这种细粒度。而低频维度编码的是全局位置，波长超过 L，在训练时连一个完整周期都没走完。这些维度面对更长的序列时，才真正需要通过 PI 进行“插值外推”。NTK-by-parts 完美地用数学将这一直觉落地。

5. 注意力缩放 (Attention Scaling)

YaRN 发现，在插值后，注意力的 Logits 分布会发生变动（平均长度变大导致方差改变）。为了修正这种温度偏差，YaRN 在注意力计算后乘以一个极小但关键的缩放系数：

t = ( 0.1 · ln(s) + 1 )-2
